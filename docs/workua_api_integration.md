# work.ua REST API — як підключити

Документація отримана від work.ua 2026-06-22 (`Work_API_публікація_вакансій_отримання_відгуків.docx`).
TL;DR — нема окремого API-токена. Авторизація = email + пароль акаунту роботодавця.

## Ключове за документацією

| Параметр | Значення |
|---|---|
| Base URL | `https://api.work.ua/` |
| Auth | HTTP Basic — email + пароль work.ua employer |
| Rate limit | 50 запитів/сек (IP-блок на 1 хв при перевищенні) |
| Локалізація | Header `X-Locale: uk_UA` |
| User-Agent | `<Компанія> (<email>)` — рекомендовано |
| Лоут-аут | 10 невдалих авторизацій → блок акаунту на добу |

## Що нам найважливіше

### 🟢 БЕЗКОШТОВНО (тягнемо часто)
- `GET /jobs/responses` — усі відгуки на наші вакансії
- `GET /jobs/<id>/responses` — відгуки на конкретну вакансію
- `GET /jobs/my` — наші вакансії
- `GET /dictionaries` — довідники (міста, категорії, рівні зарплати)
- `GET /available-publications` — скільки публікацій залишилось

### 🔴 ПЛАТНО (відкриває контакти!)
- `GET /resumes` — пошук в базі резюме (списує оплачені відкриття)
- `GET /resume?resume_id=N` — повне резюме (списує)

**Стратегія:** основний інбаунд — `GET /jobs/responses` (free). Якщо обʼєму
мало → доповнюємо платним `/resumes` на вузькі запити (Менеджер з продажу +
конкретний регіон + досвід).

## Що тобі робити прямо зараз

### Крок 1 — отримати креди
Тобі потрібні **email + пароль** від акаунту роботодавця Kozyr Trans на work.ua.
Це той самий логін, яким заходять у `work.ua/employer/`.

**Якщо сайт не відкривається** (як ти писав):
- Спробуй через VPN (work.ua блокує деякі IP)
- Спробуй мобільний інтернет замість Wi-Fi
- Спробуй `https://api.work.ua/dictionaries` — це публічний ендпойнт, має відкритися без логіну. Якщо й він не відкривається → проблема з провайдером.
- Якщо API не відкривається з твоєї мережі → доведеться запускати з VPS (Hetzner Helsinki). API ходитиме з європейського IP.

### Крок 2 — перевірити що API увімкнено для акаунту
Зателефонувати на саппорт work.ua або написати листа партнерам:
> Доброго дня! Підтвердіть, будь ласка, що для нашого акаунту [email]
> увімкнено API-доступ за документом який ви надіслали 22.06. Хочемо
> зробити перший тестовий запит на `GET /dictionaries`.

### Крок 3 — передати креди безпечно
**НЕ в чат**. Варіанти:
1. **Bitwarden Send** (легко) — bitwarden.com/products/send
2. Створи окремого менеджера на акаунті (підлеглого від адміна) спеціально під API:
   - `api@kozyr-trans.com` або подібний
   - підтверди email
   - пароль = згенерований випадковий
   - якщо щось піде не так — відкличеш доступ цього юзера, не чіпаючи свій основний

### Крок 4 — тест через curl
Коли отримаєш креди, я можу тебе провести через перший живий запит з твого
Mac'а (терміналу), щоб переконатись що працює:
```bash
curl -i --basic -u 'EMAIL:PASSWORD' \
     -H 'User-Agent: Kozyr Trans (EMAIL)' \
     -H 'X-Locale: uk_UA' \
     https://api.work.ua/jobs/my
```
- `200 OK` + JSON зі списком вакансій → працює
- `401` → неправильний пароль / API не активований
- `403` → акаунт заблоковано → дзвонити саппорту
- `429` → почекати 1 хв і повторити
- `Connection timeout` → у тебе мережа блокує API → використовуємо VPS

### Крок 5 — підключаємо в проект
Як тільки працює curl — записуємо креди в `.env`:
```
WORKUA_EMPLOYER_EMAIL=...
WORKUA_EMPLOYER_PASSWORD=...
```

Далі мій код у `src/integrations/workua_api.py` все робить сам:
- кешує довідники (`/dictionaries`)
- кожні 5 хв тягне нові відгуки (`/jobs/responses?last_id=<останній>`)
- штовхає їх через `InboundRouter` → `KeyCRM` з дедупом

## Що буде ходити з нашого боку

### Кожні 5 хвилин (free, інбаунд)
```
GET /jobs/responses?last_id=<cursor>&limit=50&from_type[]=send&from_type[]=phonecall
```
Cursor зберігаємо в `.cache/workua_cursor.json`. Тільки нові з минулого опитування.

### Раз на день (free)
```
GET /jobs/my?full=1
GET /dictionaries          # тільки якщо кеш > 24h
GET /available-publications
```

### За запитом (платно, тільки коли мало інбаунда)
```
GET /resumes?search=...&region[]=<id>&phone=1&period=3&limit=20
GET /resume?resume_id=<id>
```
Це включаємо обережно, з лімітом на день, щоб не випалити пакет контактів.

## Безпека

- Пароль НІКОЛИ не в гіті. Тільки в `.env.enc` через sops + age-key
- Окремий API-юзер (не основний адмін) — щоб у разі компромету відкликати без болю
- На VPS файл `.env` має право 600, власник — service user
- При перших ознаках бану акаунту — зупинити сервіс, дзвонити менеджеру work.ua

## Інше з документу (на майбутнє)

- `POST /jobs` — можемо публікувати/оновлювати вакансії з нашого CRM
- `PUT /jobs/<id>/close` — деактивувати після закриття позиції
- `DELETE /jobs/<id>` — видаляти
- `text_response=1` — дозволити відгук без резюме (це для нашої воронки погано — менше даних, але більше об'єму)
- `show[phone]=1` — публікувати наш телефон у вакансії
- `publication=standart_job` — тип публікації

Ці методи не потрібні для MVP, але код-клієнт уже їх підтримує.
